# Attacker-driven PyRIT

PyRIT is the exploitation stage and consumes qualified garak handoff objectives. `PromptSendingAttack` remains available only as a labelled single-turn baseline. Multi-turn follow-ups are generated by the independent attacker role through the model gateway and include the authorized objective, sanitized untrusted target response, expected boundary, and remaining budget. The target remains the customer model and the judge is a separate role.

Evidence is considered attacker-driven only when an official PyRIT attack executes, the attacker invocation is recorded, at least two target turns occur, a later prompt depends on an earlier response, and native PyRIT memory/results artifacts exist. Otherwise the result is labelled `single_turn`, `limited`, or `failed` and is never presented as a successful multi-turn run.

Execution stops on deterministic telemetry success, judge-supported candidate success, stable refusal, duplicate prompt, budget exhaustion, or the operator kill switch. Retrieval, tool, confirmation, and memory outcomes are decided by native telemetry; the judge can create only a candidate and cannot override deterministic telemetry.
