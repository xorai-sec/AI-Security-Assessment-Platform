# Security Policy

This platform is for authorized AI security assessment only.

Do not use it against public or third-party systems without written authorization.
Do not submit real personal data, secrets, production credentials, or confidential
customer records into the demo target.

Report security issues privately to the repository owner. Do not create public
issues containing exploit details, credentials, tokens, or sensitive evidence.

The current first release stores sanitized evidence on local disk for laboratory
demonstration. Production deployments should configure encrypted object storage,
central audit logging, and access-controlled report distribution.
